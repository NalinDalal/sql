# DataBase Installation
For now we will introduce db locally in a container via docker


1. Pull the Postgres Database Image -> `docker pull postgres`
2. Run the Container-> 
`docker run -d \
  --name postgres-db \
  -p 5432:5432 \
  -e POSTGRES_PASSWORD=YourPassword \
  postgres
`

3. Access the Database Inside the Container-> `docker exec -it postgres-db bash`

Inside the container, connect to PostgresSQL:`psql -U postgres`

4. Create a User and Start Writing PL/SQL
`CREATE USER testuser WITH PASSWORD 'testpassword';
CREATE DATABASE testdb OWNER testuser;
`

5. Log in as the new user(testdb): `psql -U testuser -d testdb`

## TableSpaces
database comprises of a number of physical files called data files.
files are logically grouped together into an Oracle (logical) structure called a Tablespace.

1. Command to create DataBase:
`CREATE DATABASE `

2. CREATE Command for TABLESPACES
directory for tablespace:
`sudo mkdir -p /var/lib/postgresql/my_tablespace
sudo chown postgres:postgres /var/lib/postgresql/my_tablespace
`

```sql 
CREATE [TEMPORARY] TABLESPACE DATAFILE/TEMPFILE '<Path and Filename>'
[AUTOEXTEND [OFF/ON NEXT <integer>[K/M]
MAXSIZE [<integer>[K/M]/UNLIMITED] ] ]
[LOGGING/NOLOGGING] [TEMPORARY/PERMANENT] [OFFLINE/ONLINE]
DEFAULT STORAGE <storage_clause> [MINIMUM EXTENT <integer>[K/M]]
[EXTENT MANAGEMENT [DIRECTORY/LOCAL
_ [AUTOALLOCATE/UNIFORM [SIZE <integer>[K/M] ]] ] ]
[CHUNK <integer>] [NOCACHE]
```
